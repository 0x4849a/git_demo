git clone -- given a url, it makes a local copy of a repo
git branch -- allows us to keep code separated based on what we are trying to develop
git branch -a -- shows us the existing branches on our repo
git remote -v -- shows us the url/repository that our current repo is aware of
git add -- lets us add new files to be tracekd by git to our repo
git commit -a -- saves all of our changes on local computer into a commit(a set of changes)
git commit file1 file2 ... -- commits only specific file changes to our local branch
git push -- this will send our local commits to our default remote git service
git pull -- check the remote server for new commits. If there are, it will fetch (git fetch) and merge them with existing code in local branch (git merge).
